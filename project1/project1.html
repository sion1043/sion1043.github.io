Steam 플랫폼 게임 성공요인 분석 및 시각화

Python SQL Selenium Databricks Power BI

기간: 2025. 05 - 2025. 06 (약 1개월)

역할: 데이터 수집·분석 파이프라인 구축, Power BI 대시보드 시각화

프로젝트 목표

  • 게임 개발사 대상 리뷰 관리 및 운영 지원 B2B 대시보드 제공
  • Steam 리뷰 데이터 분석을 통해 게임의 성공 요인 도출

프로젝트 배경

  • Steam 리뷰는 게임 품질·만족도·문제점·인기 등을 실시간으로 반영합니다.
  • 리뷰 점수와 개수는 상점 노출·추천 알고리즘·판매량과 직결됩니다.
  • 유저 리뷰 데이터 관리는 게임의 성공과 지속성에 중요한 역할을 합니다.

프로젝트 필요성

  • 개발사와 유저 간 소통 채널을 확보합니다.
  • 리뷰 데이터 인사이트 도출로 성공 가능성을 향상시킵니다.
  • 리뷰 기반 게임 개선 및 업데이트 방향성을 지원합니다.
  • 리뷰 반응 추적을 통한 유저 니즈를 파악합니다.

서비스 차별점

  • 리뷰 수정율, 위험 리뷰, 유저 이탈 분석 등 성공/실패 패턴을 직관적으로 볼 수 있는 지표를 설계했습니다.
  • 크롤링 + API 결합으로 대용량의 데이터를 통한 신뢰성을 확보했습니다.
  • Power BI 대시보드로 시각적 직관성과 실시간 탐색성을 강화했습니다.
  • 게임사·퍼블리셔가 전략적으로 활용할 수 있는 인사이트를 제공합니다.
  • 데이터 불균형을 보완하는 기간 슬라이서와 대시보드 상호작용 기능을 구현했습니다.
  • 마케팅 팀이 없는 소규모 개발사도 인사이트를 얻고 보완할 수 있습니다.

데이터 수집

  • Steam API와 Selenium을 활용했습니다.
  • RPG 인기 게임 200개와 최고 평점 200개를 **Top-tier**로 선정했습니다.
  • 랜덤 샘플 1,000개를 **Other 게임**으로 선정했습니다.
  • 최종적으로 1,037개 게임의 리뷰를 200건씩 수집했습니다.

데이터 전처리

  • 결측치 처리 (프로필 비공개, 플레이시간, 리뷰 작성 당시 플레이시간 결측)
  • 비유효 데이터 삭제 (정식/얼리액세스 출시일 이전 리뷰 제거)
  • 이상치 처리 (마지막 플레이 일자 1970년 1월 1일 → 삭제)
  • 파생컬럼 생성
  • 리뷰 텍스트 전처리: 영어/숫자 이외 제거, 이모지/URL/HTML/특수문자/구두점 제거, 유니코드 표준화, 공백 정리, 불용어 처리 등을 수행했습니다.

데이터 분석

  • 분석과 통계를 통해 전체 게임 중 상위 게임을 선정했습니다. 상위 게임은 추천 리뷰 비율이 77% 이상인 게임으로 정의했습니다.
  • 날짜별 리뷰 개수 분포, 출시년도, 가격, 할인율 등을 분석했습니다.
  • 유저 리뷰 커뮤니케이션 활성도(답글, 반응, 좋아요/싫어요)를 분석했습니다.
  • 상위 게임과 일반 게임을 비교하여 리뷰 수정율, 리뷰 답변 수, 2주간 플레이 시간 등 컬럼을 비교했습니다.
  • 상위 게임은 긍정적 컬럼 비율이 높고, 부정적 컬럼 비율이 낮다는 결과를 확인했습니다.
  • 샘플 불균형으로 인해 등분산성이 위배되고 정규성 가정이 충족되지 않아 비모수 검정을 수행했습니다.
  • 반복 검증을 통해 다른 조건에서도 동일한 경향을 확인했습니다(수치형, 범주형, 시계열).
  • 리뷰 기반 클러스터링을 통해 RPG 장르의 KPI는 높은 내러티브 몰입도임을 도출했습니다.
  • 허깅페이스 GTE Large 모델을 적용했습니다. (데이터셋 링크)
  • 샘플 부족 문제를 해결하여 키워드 9개에서 7개로 줄였고, Macro F1 점수를 63%에서 87%로 향상시켰습니다.

문제 상황 발생 및 해결

1. 데이터 선정 문제

문제: E-commerce 데이터 부족 및 출처 불명으로 데이터셋 선정에 어려움이 있었습니다.

해결: Steam 플랫폼 데이터를 활용하여 웹 크롤링 및 API로 리뷰를 수집, 사용자 피드백 기반 분석이 가능해졌습니다.

2. 작업 효율 및 역할 분배 문제

문제: 다양한 접근 시도로 인해 프로젝트 진행이 지연되었습니다.

해결: 수집팀과 분석팀으로 역할을 분리하여 데이터 확보와 방향성 도출에 집중했습니다.

3. 데이터 수집 중 문제

문제: Steam API 요청 제한, 이상치/공백/ASCII 그림 리뷰 등 예외 데이터가 존재했습니다.

해결: 요청량 및 슬립 시간을 조절하여 게임 약 17,000개, 리뷰 약 22만 건을 확보했습니다. 이상치와 결측치를 제거하고 포맷을 정리했습니다.

4. 성공/비성공 게임 정의 문제

문제: 성공/비성공 게임의 명확한 기준을 선정하기 어려웠습니다 (Steam Awards, 논문, 성과 지표 등).

해결: 최고 수익 및 최고 평가 게임을 성공 게임으로 선정하고, 비교 대상으로 무작위 게임을 선택하여 분석을 통해 기준의 신뢰성을 확보했습니다.

5. 모델링 성능 문제

문제: 로컬 하드웨어 제한으로 F1 점수가 저하되었습니다.

해결: Colab Free를 활용해 일부 분류에 성공했고, 전체 리뷰는 Databricks ML GPU 클러스터를 활용하여 전체 데이터 분류에 성공했습니다.

6. 수집 데이터 불균형

문제: 게임당 리뷰 200개로 고정 수집하여 리뷰 기간의 불균형이 발생했습니다.

해결: 대시보드에 기간 슬라이서를 적용했고, 추후 리뷰 데이터 확장이 가능하도록 설계했습니다.

기대 효과

  • 게임 개발사는 리뷰 기반 KPI를 파악하고 개선 전략을 수립할 수 있습니다.
  • 유저는 피드백이 반영된 게임 개선으로 만족도가 상승할 것입니다.
  • 프로젝트 결과물인 분석, 모델, 대시보드를 일괄 제공하여 B2B 서비스로 구현될 수 있습니다.
